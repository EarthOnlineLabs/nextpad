多个会话并行时，灵感冒出来，却记不清这条该发给哪个会话。Nextpad 把便签按会话自动绑定——不用建文件夹、不用打标签，切换会话时它自己跟着换。省去你在飞书自聊里手动分类、还总是发错地方。

01

三个亮点

⤵

零点击自动跟随

切换 Claude 会话，便签内容自动跟着切，几乎瞬时跟手。不用点、不用选，这是它存在的全部理由。

~10ms 响应
●

一个字都不读

只读取「当前是哪个会话」的一个编号，从不打开你的对话记录。全程本地、开源，代码 700 行，自己就能看懂。

仅 session ID
◇

小巧原生

原生 macOS App，不是 Electron。无账号、无后台、卸载不留痕。一枚安静浮在屏幕角落的小药丸。

~1MB · 无需 Apple 账号
02

怎么用

安装步骤

① 下载   点这里下载 Nextpad.zip  Safari 会自动解压出 Nextpad.app。

② 移到「应用程序」   把下载到的 Nextpad.app 拖到「应用程序」文件夹。

③ 双击打开   系统会弹出安全提示，点「完成」关掉它——这一步是正常的。

④ 放行   打开 系统设置 → 隐私与安全性，往下滚到底部，会看到「已阻止 Nextpad」，点旁边的 「仍要打开」，输入密码确认。

放行后 Nextpad 自动启动，浮窗出现在屏幕右上角。

或者：从源码编译（粘给 Claude 一段指令即可）
粘给 Claude 一次性安装 
帮我在这台 Mac 上安装并运行 Nextpad —— 一个浮在屏幕角落、自动跟随我当前 Claude 会话的便签小工具（开源，只读「当前会话编号」，不读任何对话内容）。请全自动完成，遇到下面的已知情况自行处理，装好后用一句话告诉我怎么用：

1. 确认有 Swift 编译环境（命令 swiftc）；没有就先装：xcode-select --install（会弹安装窗，我点同意）。
2. 克隆源码并编译运行：git clone https://github.com/EarthOnlineLabs/nextpad ~/.nextpad （已存在就 cd ~/.nextpad && git pull），然后 cd ~/.nextpad && ./build.sh run
3. 若编译报错提到 SwiftBridging 或重复的 module.modulemap：按 build.sh 打印的提示执行那条 sudo mv 命令（会让我输一次 Mac 登录密码），再重新 ./build.sh run
4. 确认 Nextpad 浮窗出现在屏幕右上角。
5. 复制 ~/.nextpad/build/Nextpad.app 到「应用程序」，并设为登录时自动启动。

全程本地编译，不需要任何 Apple 账号或证书。

装好之后，日常就四件事

记记一条：在输入框打字，回车，存好。
跟自动跟随：切到另一个 Claude 会话，便签内容自动跟着换——零操作。
用用一条：点一下卡片 = 自动复制，回 Claude 粘贴发送。
收收起：点右上角小横杠缩成小药丸，点药丸再展开。
